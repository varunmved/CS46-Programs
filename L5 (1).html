In this assignment you will be working with standard input, standard output, and outputting to files.

Before You Begin

Update your VM to get the latest packages needed for this assignment.

$ appliance upgrade all

Fork and Clone the Repository

You will use Git and BitBucket to both start and submit the project. The first step is to fork the starter repository on BitBucket, then clone it into your virtual machine.

  1. Log into BitBucket and go to the repository at https://bitbucket.org/profbbrown/lab-5-files
  2. Fork the repository. Click on the three dots in the left sidebar and select Fork. On the next screen, check the box labeled "private repository" and click Fork Repository. This will create a copy of my repository in your BitBucket account.
  3. Clone the repository. Click the three dots in your new (forked) repository and select Clone. Copy the command out of the box.
  4. Go to your VM and paste the command. This will pull a copy of your BitBucket repository to your VM. It will automatically create a directory to hold the working directory.

You will now have a directory containing two files: homelistings.csv and homes200.csv. The first file contains 9977 homes for sale in Sacramento and Placer counties. The second file is 200 random lines from the first file. You can use the smaller file for testing with a smaller set of homes.

Each line of the file consists of the following fields:

  • zip code
  • ID
  • address
  • price
  • number of beds
  • number of baths
  • area

Problem 1: Convert to JSON

In this program you will convert the CSV file to JSON. Only the ZIP code, address, price, and area will be in the JSON file. It will look like this:

[
{ "zip": 95678, "address": "123 Main St.", "price": 234000, "area": 1700 },
{ "zip": 95815, "address": "1000 Elm St.", "price": 250000, "area": 1500 },
...
{ "zip": 95677, "address": "222B Baker", "price": 399000, "area": 1850 }
]

Your program, contained in homes2json.c, will read from standard input and write to standard output. You can run it like this:

./homes2json < homelistings.csv > homes.json

After running the program, you can view the resulting JSON file to see that it has the correct format. Things to watch for:

  • The attribute names (zip, address, price, and area) must be in quotes.
  • The address itself must also be in quotes.
  • There is no comma after the last home.

Did you produce valid JSON?

The jsonlint program can tell you if you have a properly formatted JSON file. Run it like this:

$ jsonlint -v homes.json

If it says "ok", great! But if not, it will tell you where you have a problem.

Problem 2: Calculate Home Prices

Write a program in homeprice.c that calculates three values: the cheapest home, the most expensive home, and the average price of all homes. Your program doesn't need to actually open any files for reading. Just get input using scanf like you always have and run the program using redirection:

./homeprice < homelistings.csv

Your program should output just the three numbers separated by a space each and terminated with a newline. Only three integers. No extra output.

Running the program on the homes200.csv file should produce the following output:

./homeprice < homes200.csv
59900 8000000 416579

Problem 3: Small, medium, large

Read the homelistings.csv file from standard input using the techniques we learned in class and in the textbook. You will write the data to three files: small.txt, med.txt, and large.txt.

A small home is less than 1000 sq ft. A medium home is 1000-2000 sq ft. And a large home is more than 2000 sq ft.

To each file, write the address and area of the home, separated by a colon, like this:

123 Main St. : 1734
7842 Sunset Blvd. : 3910

Verify your program is working by inspecting each of the output files using an editor. Make sure the areas of all the homes in the small.txt file are less than 1000 sq ft, all the homes in the med.txt file are between 1000 and 2000 (inclusive), etc.

The name of this program should be sml.c (stands for small-medium-large).

Problem 4: Zip codes

Segregate the homes by zip code. Write a program to create a text file for each zip code in the file. For example, if the zip codes 12345, 95678, and 34872 are in the file, you should end up with three files: 12345.txt, 95678.txt, and 34872.txt.

The input file is read from standard input, as shown in the textbook.

To each file just write the addresses of the homes.

The name of this program should be zips.c
